# Growth mindset 

Before we get into the technical stuff, I want to talk about something more foundational. Something that matters no matter who you are and no matter what you are trying to teach.

Growth Mindset.

Who has heard of the growth mindset? 

- Definition.
- Growth versus Fixed 

So clearly it's good to promote a growth mindset in your students. What are some ways to do that?
- let them finish stuff, don't leave them behind
- show them that it's normal for stuff to be hard (in fact, when the learning is hard, it usually means it is working better)
- celebrate effort over smarts 
- show techniques for learning, rather than just the material 
- celebrate progress "you used to suck at this! now you are getting everything right!"

# Sheena's Taxonomy :P 

This just covers the very basics of code

Level 1: Reader
- Can read and understand basic code structures (return, for loops, variables, etc)
- Common challenges: Error messages, print versus return, what do loops loop over, classes, scope, imports

Level 2: Translator  
- Can translate simple words into simple code 
- Can often come up with perfectly respectable algorithms, then write down something totally different
- Can do the easiest hacker rank problems, problems that require no decomposition or serious problem solving techniques

Level 3: Problem solver 
- Can solve algorithmic problems using standard problem solving skills (decomposition, brute force, etc)
- On hacker-rank this would be medium and hard problems
- Changing random lines of code to make the code work stops being an effective strategy, no "coding with a hammer" 
- Common problem: looking at solutions and assuming they could have done it, like learning to drive through teleportation. The act of solving problems is important here. 



Level 4: Composer 
- Can translate real world problems into code, eg make a website that lets people borrow tools from each other. 
- Understands how data flows in complex applications. Eg there is a database, a backend that generates html, html and js get rendered on the client device

Level 5: The Pragmatist
- Has taste and can apply it. DRY, clean, cohesive, loosely coupled
- Eg: understands why we write tests and does that meaningfully
